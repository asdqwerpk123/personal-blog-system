## README Merge Design

日期：2026-04-01

### 目标

将主项目现有 `README.md` 与已验证的后端基础实现说明整合为一份中文、面向开发者的仓库首页文档。

### 设计原则

- 保留主项目 README 中已有的项目背景、课程定位和数据库设计说明。
- 补充工作树中已经验证通过的运行方式、测试方式、Druid 监控、MyBatis-Plus 与代码生成范围。
- 文档组织以“快速上手”优先，避免写成课程汇报材料。
- 不覆盖用户当前未提交的 `sql`、`docs` 等其他内容。

### 目标结构

1. 项目简介
2. 技术栈
3. 项目结构
4. 当前进度
5. 快速开始
6. 数据库设计说明
7. 后端基础能力
8. Druid 监控页
9. 当前代码生成范围
10. 后续开发建议

### 合并策略

- 以主项目当前 `README.md` 为基础语境。
- 将工作树中已验证的后端基础设施说明并入“快速开始”和“当前进度”部分。
- 最终输出为一份完整的中文 README。
